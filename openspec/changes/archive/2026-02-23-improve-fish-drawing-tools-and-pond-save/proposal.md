## 背景与动机

当前画板绘鱼能力过于基础，用户无法便捷地进行颜色切换、笔触调整与错误回退，导致绘制体验不足。与此同时，已完成的鱼在部分流程中未稳定进入鱼塘，破坏了“绘制-保存-可见”的核心闭环，因此需要尽快修复并完善。

## 变更内容

- 增加完整绘制工具栏：画笔、橡皮、颜色选择、笔触大小调节、撤销。
- 将“保存”设为明确提交动作，用户通过保存确认将当前绘制结果提交为鱼。
- 修复绘制后入池链路，确保保存成功的鱼稳定加入鱼塘/鱼池并在后续流程可见。
- 使用产品指定图标资源：
  - 保存：`https://turing-test-1319469298.cos.ap-guangzhou.myqcloud.com/save.png?imageSlim`
  - 橡皮：`https://turing-test-1319469298.cos.ap-guangzhou.myqcloud.com/eraser.png?imageSlim`
  - 画笔：`https://turing-test-1319469298.cos.ap-guangzhou.myqcloud.com/draw.png?imageSlim`

## 能力范围

### 新增能力

- `fish-drawing-toolkit`：定义画板工具能力（画笔/橡皮切换、色板、笔触大小、撤销）及工具栏交互行为。
- `fish-drawing-save-flow`：定义显式保存语义与保存后入池一致性，保证保存成功的鱼进入鱼塘可见状态。

### 修改能力

- 无。

## ui节点

- 应在应该使用的 地方 使用 widget保持定位关系

## MCP

在本变更中优先使用 cocos-mcp 辅助完成场景节点检查与工具栏接线验证。

## 影响范围

- 影响画板与多人模式鱼创建相关 UI/逻辑代码。
- 影响绘制结果提交到鱼塘状态的数据链路。
- 工具栏依赖远程图标资源加载与兜底策略。
- 需要补充/更新 scene-contract 与行为测试，覆盖工具能力与保存入池可靠性。
